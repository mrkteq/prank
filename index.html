Portfolio: AI-Directed Design

Demonstrating Strategic AI Collaboration Through Intentional Anti-Design

About This Project

The Concept

What you just experienced was not an accident‚Äîit was a carefully orchestrated demonstration of AI prompting, direction, and collaboration skills. Every terrible design choice, clashing color, and Comic Sans heading was intentionally crafted through strategic prompts to an AI language model.

Why This Matters

Building something "bad" on purpose is significantly harder than building something good by accident. This project showcases:

  • The ability to give clear, specific creative direction to AI systems
  • Strategic thinking about user experience (even when deliberately breaking it)
  • Technical implementation skills with HTML, CSS, and JavaScript
  • Understanding of design principles (demonstrated by intentionally violating them)
  • Iterative refinement through prompt engineering

The Process

This website was created through a multi-step collaboration with Claude (Anthropic's AI assistant):

Step 1: Strategic Planning

  • Defined core objectives for the "terrible" website
  • Identified essential sections and their purposes
  • Planned the transformation reveal mechanism

Step 2: Design Specification

  • Created a comprehensive design document detailing every intentional "flaw"
  • Specified the terrible font pairing (Comic Sans + Papyrus)
  • Documented anti-UX patterns and interaction behaviors
  • Outlined the professional version requirements

Step 3: Implementation Direction

  • Prompted AI to build the complete single-page application
  • Ensured both terrible and professional versions were functional
  • Implemented smooth transformation mechanism
  • Tested responsiveness and cross-browser compatibility

Key Prompting Techniques Used

  • Specific Direction: Instead of "make it look bad," I specified exact anti-patterns like "Comic Sans for headings, Papyrus for body text, skill bars exceeding 100%"
  • Structured Documentation: Created a formal specification document that an AI could parse and execute
  • Iterative Refinement: Built the project in stages‚Äîstrategy, specification, then implementation
  • Clear Success Criteria: Defined what "successfully terrible" meant with concrete examples
  • Meta-Awareness: Designed the project to be self-documenting and self-referential

Technical Implementation

Features Demonstrated

  • CSS Animation: Spinning text, bouncing buttons, blinking elements, marquee scrolling
  • Responsive Design: Works across devices (intentionally breaks in funny ways on mobile)
  • State Management: JavaScript toggle between terrible and professional modes
  • Single Page Application: All content in one HTML file with conditional display
  • Accessibility Considerations: Even "terrible" design maintains semantic HTML structure

Design Choices Explained

  • Comic Sans + Papyrus: Two of the most mocked fonts in design, paired together for maximum effect
  • Clashing Colors: Hot pink, lime green, electric blue‚Äîcolors that create visual tension
  • Animated Elements: Everything moves to create sensory overload
  • Anti-UX Patterns: Buttons that move on hover, impossible forms, broken counters
  • 90s Web Aesthetic: Under construction GIFs, hit counters, Netscape references

What This Demonstrates

AI Collaboration Skills

This project proves I can effectively collaborate with AI systems to achieve specific creative and technical outcomes. The ability to articulate vision, provide structured direction, and iterate on results is increasingly valuable as AI tools become central to development workflows.

Design Understanding

You can't intentionally break design rules effectively unless you understand them deeply. Every "mistake" on the terrible version was a deliberate choice that demonstrates knowledge of:

  • Typography and hierarchy
  • Color theory and contrast
  • User experience principles
  • Web accessibility standards
  • Modern design trends

Technical Versatility

The project showcases proficiency with:

  • HTML5 semantic structure
  • Advanced CSS (animations, gradients, transforms, grid, flexbox)
  • JavaScript DOM manipulation
  • Responsive design techniques
  • Performance optimization (despite appearing broken)

Creative Problem-Solving

Creating a deliberately bad experience that's still functional, entertaining, and serves a portfolio purpose requires creative thinking about constraints, audience, and execution.
